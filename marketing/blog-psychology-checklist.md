# Blog Psychology Checklist

이 파일은 /write-blog 파이프라인의 1·3·5·6단계에서 참조한다.

## A. 검색 의도 분류 (1단계 사용)

키워드를 받으면 아래 6가지 중 하나로 분류한다.

- informational      : 개념·정의를 알고 싶다
- problem-solving    : 지금 겪는 문제를 해결하고 싶다
- comparison         : A vs B를 비교하고 싶다
- purchase-intent    : 맡길지 직접 할지 결정 직전
- troubleshooting    : 시도했는데 안 된 원인을 찾는다
- validation         : 자기 판단이 맞는지 확인받고 싶다

## B. 독자 심리 5요소 (1단계 출력에 반드시 포함)

1. currentProblem    : 지금 겪는 구체적 불편
2. emotionalState    : 답답함 / 불안 / 의심 / 피로 / 비용부담 중 1~2개
3. desiredOutcome    : 정보가 아니라 더 나은 상태
4. failedAttempts    : 이미 시도했지만 실패했을 법한 것
5. likelyObjection   : 글을 읽고 가질 만한 거부감

## C. 도입부 규칙 (3·5단계)

첫 200자 안에 아래 흐름이 보여야 한다.

- 첫 문장: currentProblem을 그대로 짚는다
- 두 번째 문장: 대부분이 놓치는 지점
- 세 번째 문장: 글 전체에서 무엇을 다룰지

금지:
- "오늘은 ~에 대해 알아보겠습니다"
- "안녕하세요"
- 서비스 자기소개로 시작
- 키워드 정의로 시작 (informational 의도일 때만 허용)

## D. 본문 톤 (5단계)

- 가르치려 들지 않는다
- 독자를 탓하지 않는다
- 과장하지 않는다 (배수 표현, "최고", "보장" 금지)
- "우리 서비스가 최고"보다 "이 문제는 이렇게 해결해야 한다"를 먼저
- 근거 없는 수치 금지

## E. 서비스 연결 (5단계)

❌ "그래서 우리 서비스를 이용하세요"
✅ "이 과정을 직접 정리하기 어렵다면, ___부터 ___까지 함께 진행하는 방식도 고려할 수 있습니다"

## F. CTA 강도 매트릭스 (1단계에서 ctaStrength 결정)

| searchIntent       | ctaStrength    |
|--------------------|----------------|
| informational      | weak           |
| problem-solving    | medium         |
| comparison         | medium-strong  |
| purchase-intent    | strong         |
| troubleshooting    | medium         |
| validation         | medium         |

## G. 필수 문장 흐름 (5단계 본문 어딘가에)

> 독자는 ___ 때문에 검색했다.
> 하지만 대부분 ___만 신경 쓰고, 정작 ___은 놓친다.
> 이 문제를 해결하려면 먼저 ___을 확인해야 한다.

## H. 글 길이 가이드 (3단계)

- informational     : 2000~3500자
- problem-solving   : 4000~6000자
- comparison        : 5000~8000자
- purchase-intent   : 5000~8000자
- troubleshooting   : 3000~5000자
- validation        : 3000~5000자

길이 채우려고 filler 추가 금지.
