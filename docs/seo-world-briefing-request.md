# [작업 지시] SEO월드 광고 기획 브리핑 생성

> 이 프롬프트는 **SEO월드 프로젝트 디렉토리에서 Claude Code 세션을 실행한 뒤 붙여넣어 실행**한다.
> 목적: SEO월드의 **실제 코드/콘텐츠를 근거로** 광고 기획용 브리핑 문서를 생성하는 것.
> 최종 산출물: `prompt/seo-world-briefing.md` 1개 파일.

---

## 작업 배경

나는 SEO월드 프로젝트에 Google Ads 검색광고를 새로 설계해서 집행하려 한다. 이후 이 브리핑을 별도 Claude 세션에 입력해서 캠페인/광고그룹/키워드/광고 카피 설계안을 받을 예정이다. 따라서 이 브리핑은 **"다음 AI 가 광고를 잘 설계할 수 있을 만큼 정보가 충분하고 정확해야"** 한다.

## 네가 할 일

SEO월드 프로젝트를 정밀 탐색해서 아래 6개 섹션을 가진 브리핑을 `prompt/seo-world-briefing.md` 에 저장한다.

**매우 중요한 규칙**:
- **코드에서 확인된 사실만 기입**. 추측 / 인상 / 느낌 금지.
- 코드로 확인 불가한 항목은 반드시 `[사용자 확인 필요: 무엇을 채워야 하는지]` 형태로 placeholder 를 남긴다.
- "아마도", "추정컨대", "일반적으로" 같은 모호한 서술 금지.
- 한국어로 작성.

---

## 탐색 순서 (이 순서 그대로 따라라)

### Step 1. 프로젝트 정체성 파악
다음 파일을 먼저 읽는다:
- `package.json` → 프로젝트명, 주요 의존성, 스크립트
- `README.md` / `README.ko.md` 등 → 프로젝트 소개
- `CLAUDE.md` 존재하면 → 프로젝트 지침
- `public/index.html` 또는 `app/layout.tsx` / `src/app/layout.tsx` → meta 태그 (title, description, keywords)
- 홈 페이지 라우트 (`src/app/page.tsx` 또는 `pages/index.*`) → 메인 메시지

추출: **"SEO월드는 무엇이고, 어떤 문제를 해결하며, 누구를 위한 서비스인가"** — 3줄 이내 요약.

### Step 2. 무료 도구 리스트 추출
라우트 구조를 탐색해서 도구 목록을 만든다:
- `src/app/**/page.tsx` 또는 `pages/*.tsx` / `app/tools/*` 등
- 동적 라우트라면 도구 메타데이터가 어디 저장되어 있는지 (DB schema / JSON / 상수 파일) 찾는다
- 각 도구의 **이름**, **한 줄 설명**, **URL 경로**, **주요 입력/출력 개념** 추출

도구가 10개 넘으면: **랜딩 페이지에서 카테고리화되어 있는지** 확인하고 카테고리별로 정리한다.

### Step 3. 타겟 고객 파악
다음에서 근거를 찾는다:
- 랜딩 페이지 헤드라인 / 서브헤드라인 / 가치 제안 문구
- FAQ 섹션이 있다면 질문들의 어조
- 가격 페이지 / 문의 페이지의 표현
- `<meta name="keywords">` / OG 태그

**1문장**으로 타겟 고객 정의. 근거 문구를 `근거: ...` 로 병기.

### Step 4. 현재 광고 관련 코드 흔적 찾기
다음을 탐색:
- **GA4 / GTM 스크립트**: `_app.tsx`, `layout.tsx`, `public/` 등에서 `gtag`, `GTM-`, `G-` 식별자
- **gtag 이벤트 호출**: `gtag('event', ...)` 로 검색. 현재 추적 중인 이벤트명 전부 나열
- **UTM 파라미터 처리 코드**: `utm_source`, `utm_campaign` 등
- **픽셀 코드** (Meta/Kakao/Naver): 현재 설치된 픽셀 전부
- **기존 랜딩 페이지 히어로 카피 / CTA 문구**: 광고 카피 초안 재료로 활용됨
- **문의 폼 엔드포인트**: `/api/contact`, `/api/inquiry` 등 있다면 경로와 제출 성공 후 동작

### Step 5. 기존 전환 이벤트 정의 확인
Step 4 에서 찾은 `gtag('event', 'XXX', ...)` 호출을 모두 나열하고, 각 이벤트가 **어떤 사용자 행동에서 발생하는지** 호출 위치 코드로 추적해서 1줄 설명 붙인다.

예시 출력 형태:
```
- contact_submit → src/app/contact/ContactForm.tsx:45 의 handleSubmit 성공 시 호출
- tool_used → src/components/tools/ToolResult.tsx:12 의 결과 렌더 직후 호출 (파라미터: tool_id)
```

### Step 6. Google Ads 대시보드 정보는 [사용자 확인 필요] 로 남긴다
Claude Code 는 Google Ads 대시보드에 접근할 수 없다. 따라서 이 섹션은 placeholder 만 만들고, 사용자가 나중에 Google Ads 에서 복사해서 채울 수 있도록 구조만 마련한다.

## 최종 출력 포맷 (`prompt/seo-world-briefing.md` 내용)

```markdown
# SEO월드 광고 기획 브리핑

> 작성일: {{YYYY-MM-DD}}
> 목적: Google Ads 검색광고 캠페인 설계 AI 세션에 입력할 정보 모음

## 1. 프로젝트 정체성
{{3줄 이내. "무엇 / 누구를 위한 / 해결하는 문제" 구조}}

**근거 파일**: {{읽은 파일 경로 3-5개}}

## 2. 무료 도구 리스트
| # | 도구명 | 한 줄 설명 | URL 경로 | 주요 검색 의도 (추측 금지, 도구 기능 기반) |
|---|---|---|---|---|
| 1 | ... | ... | /tools/... | ... |

총 {{N}}개.

카테고리화되어 있다면:
### 카테고리 A
- ...
### 카테고리 B
- ...

## 3. 타겟 고객
{{1문장 정의}}

**근거 문구**:
- "{{랜딩에서 발췌}}" — {{파일 경로:줄번호}}
- "{{또 다른 문구}}" — {{파일 경로:줄번호}}

## 4. 월 광고 예산
- 현재: [사용자 확인 필요: 최근 1개월 Google Ads 실집행액]
- 목표: [사용자 확인 필요: 앞으로 목표 월 예산]

## 5. 전환 정의

### Primary 전환 (Google Ads 입찰 기준)
- `contact_submit` — 문의 제출 (돈 버는 전환)
- 현재 코드 구현 상태: {{Step 5 결과에 따라 "구현됨 @ 경로:줄번호" 또는 "미구현 → 구현 필요"}}

### Secondary (분석용, 입찰 제외)
- `tool_used` — 무료 도구 사용
  - 현재 구현 상태: {{...}}
- 기타 감지된 이벤트: {{Step 5 에서 찾은 나머지 이벤트 전부 나열}}

### GA4 / GTM 세팅 감지 결과
- GA4 측정 ID: {{G-XXXXXX 또는 "미감지"}}
- GTM 컨테이너 ID: {{GTM-XXXXXX 또는 "미감지"}}
- 설치 위치: {{파일 경로}}

### 현재 전환 0 추정 원인 (코드 근거 기반, 추측 금지)
{{Step 5 결과에서 발견한 실제 문제만. 예: "contact_submit 이벤트가 ContactForm.tsx 에서 호출되지 않음", "gtag 스크립트가 layout.tsx 에 없음" 등. 코드에서 문제 못 찾았으면 "코드 상 문제 없음, Google Ads 측 전환 연동 확인 필요" 로 명시}}

## 6. 현재 Google Ads 세팅 덤프

### 6-1. 코드에서 추출 가능한 재료
**기존 랜딩 카피 (광고 카피 재료)**:
- 메인 헤드라인: "{{랜딩 발췌}}"
- 서브카피: "{{...}}"
- CTA 문구: "{{...}}"
- 파일 경로: {{...}}

**기존 UTM 파라미터 처리**:
{{있으면 나열, 없으면 "없음"}}

**설치된 광고 픽셀**:
{{Meta Pixel, Kakao Pixel 등 발견된 것 전부. 없으면 "없음"}}

### 6-2. [사용자 확인 필요 — Google Ads 대시보드에서 직접 복사]
- **캠페인명 리스트**:
  - [ ] 사용자가 이 자리에 채움
- **각 캠페인의 광고그룹명 + 키워드 리스트**:
  - [ ] 사용자가 이 자리에 채움
- **현재 집행 중인 광고 카피 (제목 1/2/3 + 설명 1/2)**:
  - [ ] 사용자가 이 자리에 채움
- **최근 30일 지표** (노출수 / 클릭수 / 클릭률 / CPC / 전환수):
  - [ ] 사용자가 이 자리에 채움

## 7. 다음 단계 (참고)
이 브리핑을 완성 후:
1. 섹션 4, 6-2 를 사용자가 수동 보완
2. 완성된 브리핑을 새 Claude 세션에 입력
3. 요청: "위 정보 기반으로 Google Ads 검색광고 캠페인 3-5개 구조, 광고그룹별 키워드 10-20개, 광고 카피 3종, 월 예산 분배 제안"
```

## 품질 체크 (네가 저장 직전에 스스로 검토)

저장하기 전 다음을 검증한다:
1. [ ] 모든 숫자/고유명사에 파일 경로 근거가 붙었는가?
2. [ ] "아마도" / "추정" / "일반적으로" 같은 단어가 없는가?
3. [ ] Step 5 에서 발견한 gtag 이벤트가 **하나도 빠짐없이** 나열되었는가?
4. [ ] Step 4 의 픽셀/GTM/GA4 식별자가 정확히 기입되었는가?
5. [ ] 코드로 확인 불가한 항목은 전부 `[사용자 확인 필요: ...]` 형태인가?
6. [ ] 출력 총 길이가 **500줄 이내**인가? (길면 광고 기획 AI 가 핵심 놓침)

완료되면 저장 경로와 총 줄수를 사용자에게 보고하라.
